## Quick Start

```bash
# Install dependencies
make install

# Start Neo4j (Docker)
make docker-up

# Seed sample data
make seed

# Start the application
make start
```

- **Backend:** http://localhost:8000
- **Frontend:** http://localhost:3000
- **Neo4j Browser:** http://localhost:7474

## Domain: Healthcare

### Base POLE+O Entities

These entity types are shared across all domains using the Person-Organization-Location-Event-Object model:

- **Person** — PERSON
- **Organization** — ORGANIZATION
- **Location** — LOCATION
- **Event** — EVENT
- **Object** — OBJECT

### Domain-Specific Entities
- **Patient** — PERSON
- **Provider** — PERSON
- **Diagnosis** — OBJECT
- **Treatment** — EVENT
- **Encounter** — EVENT
- **Facility** — LOCATION
- **Medication** — OBJECT

## Agent Tools

The AI agent has access to the following domain-specific tools:

- **search_patient** — Search for patients by name or ID
- **get_patient_history** — Get comprehensive history for a patient
- **check_contraindications** — Check for medication contraindications for a patient
- **find_similar_cases** — Find patients with similar diagnoses and demographics
- **get_referral_network** — Get the referral network for a provider
- **list_patients** — List Patient records with optional limit
- **get_patient_by_id** — Get a specific Patient by ID with all connections
- **execute_cypher** — Run ad-hoc Cypher queries
- **get_schema** — View the graph schema

## Agent Framework: PydanticAI

This project uses [PydanticAI](https://ai.pydantic.dev/) with tool decorators and typed dependencies. The agent is defined in `backend/app/agent.py` using `@agent.tool` decorators for each domain tool. Supports full streaming via `agent.run_stream()`.

To change the LLM model, edit the `Agent(model=...)` parameter in `agent.py`.

See `backend/app/agent.py` for the full implementation.

## Troubleshooting

### Neo4j Connection Errors

**"Neo4j is unavailable" on all endpoints**
- Verify your `.env` file has correct `NEO4J_URI`, `NEO4J_USERNAME`, and `NEO4J_PASSWORD`
- Run `make test-connection` to validate credentials
- Run `make docker-up` first, then wait a few seconds for Neo4j to start

**Extra variables in `.env`**
- The backend accepts extra environment variables, so standard Neo4j Aura `.env` exports (with `NEO4J_DATABASE`, `AURA_INSTANCEID`, etc.) work without editing

### Backend Issues

**Port already in use**
- Change `BACKEND_PORT` in `.env` (default: 8000)
- Kill existing process: `lsof -ti:8000 | xargs kill`

**API key not set**
- Set `ANTHROPIC_API_KEY` or `OPENAI_API_KEY` in `.env` depending on your agent framework

### Frontend Issues

**Build errors after `npm install`**
- Delete `node_modules` and `package-lock.json`, then run `npm install` again
- Ensure Node.js 18+ is installed
